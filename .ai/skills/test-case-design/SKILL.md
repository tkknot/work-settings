# テストケース設計スキル

体系的で実行可能なテストケースを作成するためのスキルです。

## テスト設計技法

| 技法 | 用途 |
|------|------|
| **境界値分析** | 入力値の有効範囲の境界をテスト |
| **同値分割** | 有効・無効グループの代表値をテスト |
| **デシジョンテーブル** | 複雑な条件分岐を表形式で整理 |
| **状態遷移テスト** | 状態変化を網羅的にテスト |
| **ペアワイズテスト** | 要因組み合わせを効率的にテスト |

## テストレベル

| レベル | 対象 |
|--------|------|
| 単体テスト | 個々の関数・メソッド |
| 統合テスト | モジュール間の連携 |
| システムテスト | システム全体の機能 |
| 受け入れテスト | ビジネス要件への適合 |

## テスト観点

| 観点 | 内容 |
|------|------|
| 正常系 | 想定通りの入力で期待結果を確認 |
| 異常系 | 不正入力でエラーハンドリングを確認 |
| 境界値 | 境界付近の値での動作確認 |
| セキュリティ | 脆弱性・脅威への耐性 |
| 性能 | レスポンス時間・負荷耐性 |

## CSV出力形式

### ヘッダー

```csv
No.,ケース名,種別,テスト観点,端末観点,前提条件,期待値,備考
```

### 列定義

| 列名 | 内容 |
|------|------|
| No. | テストケース番号 |
| ケース名 | 簡潔な名称 |
| 種別 | `UI` または `実装` |
| テスト観点 | 正常系/異常系/境界値等 |
| 端末観点 | PC/モバイル/タブレット/複数 |
| 前提条件 | テスト実行前の条件 |
| 期待値 | 期待される結果 |
| 備考 | 注意事項・補足 |

### 例: ログイン機能

```csv
No.,ケース名,種別,テスト観点,端末観点,前提条件,期待値,備考
1,ログイン_正常系,UI,正常系,複数,"有効なID・パスワード","ホーム画面に遷移",
2,ログイン_異常系_未入力,UI,異常系,複数,"ID未入力","エラーメッセージ表示",
3,ログイン_異常系_パスワード誤り,UI,異常系,複数,"誤ったパスワード","エラーメッセージ表示","3回でロック"
```

### 例: APIテスト

```csv
No.,ケース名,種別,テスト観点,端末観点,前提条件,期待値,備考
1,API_GET_正常系,実装,正常系,PC,"有効なトークン","HTTP 200 + JSON","50ms以内"
2,API_GET_認証エラー,実装,異常系,PC,"無効なトークン","HTTP 401",
3,API_POST_バリデーション,実装,異常系,PC,"必須項目未入力","HTTP 400",
```

## テストケース作成のポイント

### 効果的なケース

1. **要件ベース**: 全てが要件に紐づく
2. **独立性**: 他のテストに依存しない
3. **再現性**: 誰が実行しても同じ結果
4. **明確性**: 手順と期待結果が曖昧でない
5. **保守性**: 変更に強い構造

### 避けるべきこと

- 曖昧で再現困難な手順
- 期待結果が不明確
- 個人情報をテストデータに使用
- 過度に複雑なケース
