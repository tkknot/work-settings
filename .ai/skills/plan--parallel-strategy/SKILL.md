# 並列実行アプローチの選定

Claude Code が作業を並列化する 4 つのアプローチから、タスクに最適なものを提案するためのガイドです。
出典: <https://code.claude.com/docs/ja/agents>

## このスキルを使う場面

ユーザーが **計画の進め方・タスクの進め方を相談してきた** とき。たとえば次のような相談:

- 「この作業、どう進めるのがいい？」
- 「並列でやったほうが速い？」
- 「サブエージェント使うべき？チーム組むべき？」

→ いきなり実装に入らず、まず本スキルで **どのアプローチで進めるか** を提案する。

## 4 アプローチ比較表

| アプローチ | 提供内容 | 使う場面 | 起動 |
|---|---|---|---|
| **サブエージェント** | 1 セッション内で委任したワーカーが独自コンテキストでサイドタスクを実行し、サマリーを返す | 検索結果・ログ・ファイル内容でメイン会話を埋めたくない（後で参照しない）調査・検索 | `/agents`・Agent ツール |
| **エージェントビュー** | バックグラウンドで走る複数セッションを 1 画面で監視（リサーチプレビュー） | 独立タスクを引き継ぎ、一目で状態確認し、必要時のみ介入したい | `claude agents` |
| **エージェントチーム** | 共有タスクリスト + エージェント間メッセージングを持つ複数の調整セッション。リーダーが管理（実験的） | Claude にプロジェクトを分割・割り当て・同期させたい | `/agent-teams`・`team--session` スキル |
| **動的ワークフロー** | 多数のサブエージェントを実行し結果をチェックするスクリプト。1 ターンでは大きすぎるジョブ向け | コードベース全体の監査、大規模マイグレーション、相互検証が必要な調査 | `/workflows`・Workflow ツール |

> 本リポジトリの `.claude/settings.json` では `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` と
> `enableWorkflows: true` が有効なため、エージェントチーム・動的ワークフローも利用できる。

## 選定のための質問フロー

次の 3 つの軸で絞り込む。

1. **誰が作業を調整するか？**
   - Claude が 1 つの会話内で委任と結果収集を行う → **サブエージェント**
   - 独立タスクを引き継いで後で確認する → **エージェントビュー**
   - Claude がワーカー群を計画・割り当て・監督する → **エージェントチーム**
   - スクリプトがターンバイターンの判断の代わりに計画を保持する → **動的ワークフロー**
2. **ワーカー同士の通信が必要か？**
   - サブエージェント / エージェントビューは結果を上位（会話 or ユーザー）にのみ報告する。
   - エージェントチームはタスクリストを共有し、互いに直接メッセージを送れる。
3. **同じファイルに触れるか？**
   - 触れる場合は [ワークツリー](https://code.claude.com/docs/ja/worktrees)（`/worktrees`）で作業を分離する。
   - エージェントチームはチームメイトをワークツリー分離しないため、各メンバーの担当ファイルを重複させない。

## 補助ツール（アプローチではない）

- **ワークツリー**（`/worktrees`）: 各セッションに個別の git チェックアウトを与え、ファイル編集の競合を防ぐ。
- **`/batch`**: 1 つの大きな変更を 5〜30 個のワークツリー分離サブエージェントに分割し、各々が PR を開く skill。

## 提案の出力フォーマット

相談に対しては、次の形で簡潔に提案する。

```
## 推奨アプローチ: <アプローチ名>

**理由**: <タスクの性質と 3 つの判断軸に基づく根拠（1〜3 行）>

**起動方法**: <コマンド or スキル>

**補足**: <ファイル競合があればワークツリー分離など、必要な注意点>
```

判断が割れる場合は、第 1 候補と次点を併記し、どちらを選ぶかの分かれ目を 1 行で示す。
