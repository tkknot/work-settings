# format-backlog-notation

## 概要

指定されたファイルの内容をBacklog記法に変換します。
Markdown・プレーンテキスト・その他のフォーマットをBacklog記法に変換し、BacklogのWikiページや課題詳細にそのまま貼り付け可能な形式で出力します。

## 使用方法

```
/format-backlog-notation <ファイルパス>
```

引数にファイルパスが指定されていない場合は、ユーザーにファイルパスを確認してから処理を開始する。

---

## Backlog記法 変換対応表

### 見出し

| 変換前（Markdown） | Backlog記法 |
|---|---|
| `# 見出し1` | `* 見出し1` |
| `## 見出し2` | `** 見出し2` |
| `### 見出し3` | `*** 見出し3` |
| `#### 見出し4` | `**** 見出し4` |

### 文字装飾

| 変換前 | Backlog記法 |
|---|---|
| `**太字**` または `__太字__` | `''太字''` |
| `*斜体*` または `_斜体_` | `'''斜体'''` |
| `~~打ち消し~~` | `%%打ち消し%%` |

### リスト

| 変換前 | Backlog記法 |
|---|---|
| `- 項目` または `* 項目` | `- 項目` |
| `  - ネスト` (2スペース) | `-- ネスト` |
| `    - 深いネスト` (4スペース) | `--- 深いネスト` |
| `1. 項目` | `+ 項目` |
| `  1. ネスト` | `++ ネスト` |

### コードブロック

| 変換前 | Backlog記法 |
|---|---|
| ` ```python ` | `{code:python}` |
| ` ```js ` / ` ```javascript ` | `{code:javascript}` |
| ` ``` ` | `{code}` |
| ` ``` ` （終了） | `{/code}` |
| `` `インラインコード` `` | `{code}インラインコード{/code}` |

対応言語の例: `java`, `cs` (C#), `javascript`, `python`, `ruby`, `php`, `html`, `css`, `sql`, `bash`, `xml`

### テーブル

Markdown形式:
```
| ヘッダA | ヘッダB |
|---------|---------|
| データ1 | データ2 |
```

↓ Backlog記法に変換:
```
| ヘッダA | ヘッダB |h
| データ1 | データ2 |
```

変換ルール:
- 1行目（ヘッダ行）の末尾に `h` を付ける
- セパレータ行（`|---|`）を削除する

### その他の要素

| 変換前 | Backlog記法 |
|---|---|
| `> 引用` | `> 引用` （変更なし） |
| `---` / `***` / `___`（水平線） | `----` |
| `[テキスト](URL)` | `[[テキスト:URL]]` |
| `[URL]` / `<URL>` | `[[URL]]` |
| `![alt](画像URL)` | `#image(画像URL)` |
| 改行（行内・テーブルセル内） | `&br;` |

## 実行手順

### Phase 1: ファイルの読み込みと解析

1. 引数またはユーザー入力からファイルパスを取得する
2. ファイルを読み込み、フォーマットを判定する（Markdown / プレーンテキスト / その他）
3. ファイルの構造を把握する（見出し・リスト・テーブル・コードブロックの有無）

### Phase 2: Backlog記法への変換

以下の順序で変換を実行する：

1. **コードブロックの特定**（変換前にコードブロック内の内容を保護する）
   - ` ```lang ` → `{code:lang}`
   - ` ``` ` → `{/code}`
   - コードブロック内のテキストはそれ以外の変換対象外にする

2. **見出しの変換**
   - `# ` → `* `（行頭のみ）
   - `## ` → `** `
   - `### ` → `*** `
   - `#### ` → `**** `

3. **文字装飾の変換**（コードブロック外のみ）
   - `**text**` / `__text__` → `''text''`
   - `*text*` / `_text_` → `'''text'''`（見出し変換後に処理）
   - `~~text~~` → `%%text%%`

4. **インラインコードの変換**
   - `` `code` `` → `{code}code{/code}`

5. **リストの変換**
   - `- ` / `* ` / `+ `（箇条書き）→ ネスト深さに応じて `- ` / `-- ` / `--- `
   - `1. ` / `2. `（番号付き）→ ネスト深さに応じて `+ ` / `++ ` / `+++ `
   - インデント（2スペース or 4スペース）をネストレベルに変換する

6. **テーブルの変換**
   - ヘッダ行（最初の `|...|` 行）の末尾に `h` を付ける
   - セパレータ行（`|---|---| `パターン）を削除する

7. **リンクの変換**
   - `[text](url)` → `[[text:url]]`
   - `![alt](url)` → `#image(url)`

8. **水平線の変換**
   - `---` / `***` / `___` のみの行 → `----`

9. **行内改行の変換**
   - テーブルセル内や行内の `<br>` / `<br/>` → `&br;`
   - テーブルセル内の改行（実際の改行文字）→ `&br;`

### Phase 3: 出力

1. 変換後のBacklog記法テキストをコードブロックで囲んで表示する
2. 変換できなかった要素や注意が必要な箇所をまとめて報告する
3. Backlogへの貼り付け後に表示確認することを推奨するメッセージを付ける

---

## 出力フォーマット

```
## 変換結果

以下の内容をBacklogのWiki/課題に貼り付けてください。

---
（変換後のBacklog記法テキスト）
---

## 変換の注意点（あれば）

- [変換できなかった要素や確認が必要な箇所を列挙]
```

---

## 対応していない要素

以下の要素はBacklog記法に対応するものがないため、変換をスキップして原文のまま出力し、その旨を報告する：

- 数式（`$...$`, `$$...$$`）
- 脚注（`[^1]`）
- 定義リスト
- HTMLタグ（原則として削除またはテキスト化）
- フロントマター（`---` で囲まれたYAML/TOMLヘッダ）
