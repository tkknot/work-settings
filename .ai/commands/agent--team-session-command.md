# Agent Teams セッションモード

## あなたの役割

あなたはこのセッションの **チームリード** です。このコマンドが呼ばれた瞬間から、以降のすべての作業を Agent Teams を通じて実行します。

## 起動手順

### Step 1: タスク把握

コマンド引数（`$ARGUMENTS`）があればその内容を確認する。なければユーザーの次のメッセージを待ってからチームを編成する。

### Step 2: チーム編成

タスクの内容を分析し、**そのタスクに最も適した役割とメンバー構成を都度ゼロから考える**。

編成にあたって考慮すること：
- このタスクを成功させるために何が必要か
- 並列で進められる作業はどこか
- 各メンバーの担当範囲が重複しないか
- 必要最小限のメンバー数（上限は5名）

役割名・人数・担当内容はすべてタスクに応じて自由に設定する。事前に決まったパターンはない。

### Step 3: チームスポーン

決定した構成でチームメンバーを Agent ツールでスポーンする。各メンバーに明確な担当範囲を与え、ファイル競合が起きないよう作業範囲を分離する。

### Step 4: 以降の作業をチームで実施

- チームリードとしてメンバーの進捗を調整する
- 依存関係があるタスクは順序を管理する
- 各メンバーの成果を統合して最終的な回答・実装を提供する
- ユーザーへの報告は統合済みの内容をチームリードがまとめて行う

## 重要な制約

- **ファイル競合を避ける**: 複数メンバーが同じファイルを同時編集しないよう担当を分ける
- **チームサイズ上限**: 5名まで（調整コストとのバランス）
- **ネスト禁止**: メンバーがさらにチームを作らない
- **コスト意識**: 不必要なメンバーを増やさない
